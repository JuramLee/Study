프로세스란?
    컴퓨터에서 실행중에 있는 프로그램
    CPU 스케줄링의 대상이 되는 작업과 유사한 용어로 쓰임
    하드디스크에 있는 프로그램을 실행하면, 실행을 위해서 메모리 할당이 이루어지고, 
    할당된 메모리 공간으로 바이너리 코드가 올라가게 된다. 이 순간부터 프로세스라 불린다. 
    단위(관리의 단위) 관리의 주체는 OS
    연산ㄲㅓ리, 작업

스레드란?
    프로세스 내 작업의 흐름
    프로세스 내부에는 최소 하나의 스레드(thread)를 가지고있는데, 실제로는 스레드(thread)단위로 스케줄링을 한다.


즉, 프로그램은 크롬 chrome.exe이며 이를 더블클릭하면 프로세스가 되는 것(프로그램이 실행)

프로세스가 여러개면 멀티 테스킹
스레드가 여러개면 멀티 스레딩 => 동시성 / 동기화 이슈

연산 실행한다 => 스레드 단위로
OS는 virtual memory(독립적인  제한된 공간)를 process한테 할당한다.

process에 속한 모든 thread(연산 주체)는 프로세스의 가상메모리로 공간이 제약된다.

process는 대한민국 행정상 한 가구 => 집이라는 공간(세대원) 집의 방은 각자쓴다. 방을 제외하고 다른 장소는 공용 공간
집 = virtual memory 세대원 = Thread 각자방 = thread local storage 
thread 마다 각자 고유한 Thread Lcoal storage를 갖고 각자만의 방을 갖는다(thread), 집은 프로세스 수준에게 제공된 운영체제로부터 할당받은 한정적인 공간


컴퓨터에서 전산자원이란? CPU virtual memory(RAM, HDD 1차 2차)
**ㅍ,로세스 단위 기준으로 가상 메모리가 주어진다.
OS 입장에서 프로세스 관리를 하기 위해 PCB가 생김( 관리를 위한 정보를 담는다. =>: 이 정보로 프로세스를 관리한다.ㅣ)
스레드는 프로세스에 속함(프로세스에 할당된 메모리에 자유자재로 접근할수잇다.)
프로세스에게 자원을 주는건 리눅스, 스레드 기준으로 CPU 자원을 주는건 윈도우
스레드 과ㅓㄴ리는 TCB로 한다.

CPU는 연산처리, core를 갖는다.

전처리 과정 .i
    본격적으로 컴파일 하기 전에 처리할 작업들 준비
    외부에 선언된 다양한 소스 코드 라이브러리 포함, 프,로그래밍의 편의를 위해 작성된 매크로로 변환
    컴ㅍ차일할 영역 명시


컴파일 .s
    전처리가 완료되어도 여전히 소스 코드
    전처리 완료된 소스코드를 어셈블리 언어(저급언어)로 변ㄹ환

어셈블 .o
    어셈블리어를 기계어로 변환
    목적코드를 포함하는 목적파일이 됨

    목적코드 vs 실행 파일
        목적 파일과 실행 파일은 둘다 기계어로 이루어진 파일
        목적 파일은 링킹을 거친 이후에야 실행 파일이 된다.

링킹
각기 다른 목적코드를 하나로 묶어주는 것을 링킹이라고 한다.
